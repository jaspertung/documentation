Mobile App Development

  • 2 dominant mobile app platforms: android and ios
  • Types of mobile app development
    1. native mobile apps: must use native development environment and tools unique to each platform to get full access to capabilities of native platform (best performance, but need 2 separate codebases for each app platform)
      • Android: Java or Kotlin
      • iOS: Objective C or Swift
    2. web apps: make web apps mobile-friendly to be accessed via mobile browser (easier to implement, but slower, only available through browser with internet, and less access to device features- but getting better with Progressive Web Apps (PWAs)
    3. cross-platform mobile apps: use third-party tools to write a single codebase, then use underlying technology to compile for both android and ios
      • react native: javascript, microsoft xamarin: c#, google flutter: dart
      • middle ground between native and web apps for device capabilities and speed, look and act like native apps but don't need 2 codebases
    4. hybrid apps (subset of cross-platform): combine native and web technologies to use HTML, CSS, and JS in a standalone app like a native app
      • different implementations:
      • ionic framework using WebView (embeddable browser component that runs web techs inside mobile app)
      • React Native and NativeScript frameworks that leverage JS engines to run JS outside of the browser

Intro to React Native

  • Usually runs JS code on JavaScriptCore engine for Android and iOS, then when Chrome debugging- runs on V8 engine
  • React Native Bridge- handles communication between JS side and native side
  • Expo- Expo SDK (software development kit), expo CLI (scaffold development environment), Snack, and other tools to develop React Native apps

React Native App Structure

  • AppEntry.js inside node_modules is where App is configured to be the Root Component aka the entry point of the entire application
  • App.js is main page
  • no longer using HTML and CSS (because not in browser), now use components for all UI
  • built-in components that React Native maps into corresponding native UI components for Android and iOS
  • use <View> as div equivalent and <Text> as p equivalent
  • StyleSheet api and can also make inline styles
    •                 <View style={styles.container}></View>
                const styles = StyleSheet.create({
                    container: {
                        backgroundColor: "#fff"
                    },
                })

Expo SDK

  • Expo: set of tools and services built around React Native to help develop, build, deploy, and quickly iterate apps/web apps from the same JS codebase
  • Expo development tools: Client, Expo CLI, Snack, SDK
  • Expo SDK: provides access to device and system functionality (ex. contacts, camera, GPS)

Secure Store

  • types of storage for mobile device: localStorage, AsyncStorage, Expo Secure Store, etc
  • to store small amounts of encrypted key-value storage, use Expo Secure Store API
  • 3 methods: (all have required arg of key and optional arg of options)
    1. SecureStore.setItemAsync(key, value, options): returns promise that rejects if value can't be stored on device
    2. SecureStore.getItemAsync(key, options): returns promise that resolves to stored value, or resolve to null if no entry for given key, or reject if error occurs
    3. SecureStore.deleteItemAsync(key, options): returns promise that rejects if value can't be deleted

Local Notifications

  • local notificaitons (generated from client side) vs push/remote notifications (generated from server side)
  • Expo Notifications API- access to push and local notifications, send notifications immediately or at a scheduled time (once or recurring)
    • Notifications.seNotificationHandler()- sets notification when received and app is in foreground (default: doesn't show)
    • Notifications.scheduleLocalNotificationAsync()- schedules local notification
  • need to request permission to generate notifications from underlying operating system
    • Notifications.getPermissionsAsync()
    • Notifications.requestPermissionsAsync()

Social Sharing

  • email, text, social media
  • 2 examples: Expo SDK's MailComposer API to send emails & React Native Share API to share to various apps
    • MailComposer API: MailComposer.composeAsync(options)
      • options: required argument that can take (array of recipients, strings of subject and body, boolean of whether body contains HTML, array of file attachments)
      • returns promise with status property of sent, saved, or cancelled for iOS and sent for Android (doesn't return info)
    • Share API: Share.share(content, options)
      • content: message or url or both, plus optional title & options: misc config properties

Picking an Image

  • Expo ImagePicker API- provides access to system UI to select image/video from device or take new photo/video
    • ImagePicker.launchImageLibraryAsync(options): resolved promise (if photo taken with object with media uri, width/height, media type, etc) OR cancelled property set to true (no photo taken)/li>
    • ImagePicker.launchCameraAsync(options)
    • options prop: mediaTypes (default: images), allowsEditing (default: false), aspect (aspect ratio), quality (quality of image compression), etc
  • Expo ImageManipulator API- alter image, apply transformations, configure compression level/format, if include image data in Base64 format
  • Expo Camera API- provides React component to render a preview for device's front or back camera, can adjust zoom, autofocus, white balance, flash, take photo or start/stop recording, detect faces, scan a barcode
    • more sophistcated than ImagePicker.launchCameraAsync(options)
