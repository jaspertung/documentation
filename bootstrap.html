Bootstrap Documentation

Bootstrap Grid

creates a grid layout on a page to control where elements are placed

  • need to have <meta name="viewport"> and <link rel="stylesheet" href="node_modules/bootstrap/dist/css/bootstrap.min.css"/>
  • need divs with classes: container, row, and col
    • <div class="container">
      • <div class="row">
        • <div class="col">content<div>
      • </div>
    • </div>
  • all content must be inside columns
  • container class is fixed to viewport size and only changes when hits breakpoints
    • can use container-fluid to expand to full page and changes fluidly
  • each row can have up to 12 columns
    • col-#: specifies how many columns you want
      • <div class="row">
        • <div class="col-4">4 columns<div>
        • <div class="col-8">8 columns<div>
      • </div>
    • can also use col-4 and col OR add multiple cols and bootstrap will automatically fill out the row
    • column sizes automatically adjust when viewport size changes

Breakpoints

Size Viewport Width
(XS) <576px
SM ≥576px
MD ≥768px
LG ≥992px
XL ≥1200px
  • XS is default
  • if want to have different column widths for different viewpoints can use breakpoints
    • ex. 3 columns and want middle column to be larger in medium viewport and up
    • class="col col-md-6"
    • will be 6 columns for medium, large and extra large, but 4 columns for small and extra small
    • ex. want 8 columns in large and up
    • class="col col-md-6 col-lg-8"

Bootstrap Classes

  • order-* classes: change the order of columns in a row
    • Example: order-sm-last
      • for breakpoints small and larger, the class will be ordered last in the row
  • offset-* classes: force a boostrap column to the right
    • Example: offset-1
  • jumbotron: used to call extra attention to a section that shows key messages
  • list-unstyled: removes default listitem styles (ex. bullet points)

Bootstrap Components

Color & Spacing Utility Classes

  • only small set amount of colors, need to use own CSS stylesheet to customize
  • text: use .text-__ prefix
    • Example: .text-white
  • background: use .bg-__ prefix
    • Example: .bg-info
  • margin and padding syntax: propertysides-(breakpoint)-size
    • property: m-margin and p-padding
    • sides: t-top, b-bottom, l-left, r-right
    • size: 0, 1-5, or auto
    • Example: mt-5 or pl-md-3

Navbar

Breadcrumbs

3 types

  • location: static, show where page is located in website hierarchy (ex. library -> events)
  • attribute: give info that categorizes the current page based on attributes chosen by the user (ex. rugs -> round)
  • path: show users steps taken to get to current page (confusing, least often used)

Dos and Don'ts of navigation design

  • DO use user-friendly, simple terms (ex. "About Us" rather than "Corporate Profile")
  • DO be consistent - provide same navigation throughout website
  • DO make sure that clicking on name/logo of website in header will take users back to homepage
  • DON'T use too many options (submenus if necessary, but sparingly)
  • DON'T use vague terms (ex. "Tours" instead of "Services")
  • DON'T make it difficult to users to figure out where they are in the hierarchy of your website

Bootstrap Navbar Component

  • navbar base class component: <nav class="navbar">
  • make navbar horizontal for small and up (xs will be vertical): <nav class="navbar navbar-expand-sm"> & <ul class="navbar-nav">
  • style nav items: <class="nav-item (active)">
  • style nav links: <a class="nav-link" href="#">
  • better contrast for dark background: <nav class="navbar navbar-expand-sm navbar-dark">
  • make navbar stay at the top when scrolling: <nav class="navbar navbar-expand-sm navbar-dark sticky-top">

Icon Fonts

Used to use image files for icons, now can use icon fonts or SVG icons
can style icons like normal text with CSS

  • Installing fontawesome.com (toolkit of icons)
    • npm install font-awesome@4.7.0
    • <link rel="stylesheet" href="node_modules/font-awesome/css/font-awesome.min.css">
      • additional CSS must be placed after Bootstrap CSS
  • Adding icons
    • <i class="fa"></i>
      • must add base class of fa
    • then add icon (ex. fa-home, fa-info, fa-list, fa-address-book)
    • (optional) add sizing class (ex. fa-lg)
    • (optional) style using Bootstrap (ex. text-primary) or CSS

Buttons

  • Social Buttons for Bootstrap
  • base class of btn
  • can making anchor link and icon into button
    • <a role="button" class="btn btn-link" href="tel:+12065551234"><i class="fa fa-phone"></i> 1-206-555-1234</a>
    • role="button" is html for screen readers
    • href="tel" will try to find default app to call number

Forms

  • class="form-group": default alignment of form elements (in parent element of forms)
  • class="form-control": default styling (input, textarea, and select tags)
    • can also add styles (focus states, sizing, etc)
    • .form-control-lg
  • class="col-form-label": causes labels to center vertically with associated input fields
  • class="form-check": default layout and behavior for radio and checkbox
  • class="form-check-label": align radio and checkbox label
  • class="form-check-input": default layout for radio and checkbox inputs

Tables

  • use normal html for table first (table, thead/tbody, tr, th/td)
  • then can add bootstrap table component classes
    • ex. table (default styling), table-striped, table-bordered, table-dark
  • responsive table: horizontal scroll bar just on table instead of whole page when in small viewport
    • <div class="table-responsive">

Cards

  • lightweight and flexible container to display data
  • <div class="card">
  • default takes up entire screen so can specify width or put it in a Bootstrap grid
  • default has no margins or padding
    • use class="card-body"
  • class="card-header"
    • can use div or header tags (ex. h2, h3)
  • can use Bootstrap color & spacing utility classes (bg-dark, text-white)
  • can add Bootstrap borders (ex. border-info)

Images

  • when an image's width is greater than the viewport's width, it will force the user to scroll horizontally
    • CSS: max-width: 100% (always takes up 100% of viewport) and height: auto (adjusts proportionally to width)
      OR
    • bootstrap: <img src="..." class="img-fluid">
  • .img-thumbnail: gets 1px white border and rounded corners

Media Object

  • to put image and text next to each other instead of block level element
  • in div wrapping media AND text: class="media"
  • in div wrapping text only: class="media-body"

JavaScript Components

  • JavaScript plugins have greater functionality
  • don't need to write any JS, use Bootstrap HTML custom data (data-*) attributes

HTML Custom Data-* Attributes

  • lets you create custom attributes to store any data you want in any HTML element (can be accessed later with JS or CSS)
  • data-(string of choice) (ex. data-toggle, data-target)

Navs and Tabs

  • can use Bootstrap nav classes with either ul or nav HTML elements
    • ex. simple sets of links (horizontal/vertical) or more complex tabs or pills (can add JS plugins for panes of content)
    • can use Bootstrap's built-in custom data attributes
  • ul class= "nav nav-tabs"
  • li class= "nav-item"
  • a class= "nav-link" role= "tab" data-toggle= "tab"
    • nav-link: active, disabled
    • role for dyanimc tabbed interfaces
    • data-toggle: tab, dropdown, pill
  • div class="tab-content"
  • div role="tabpanel" class="tab-pane fade (show active)" id="#"

Collapse and Accordion

  • collapse: toggles parts of a webpage in and out of view
    • highly flexible with JS plugins, data attributes, and custom JS
  • collapse + card = accordion
  • div id="acordion"
  • div class="card"
  • div role="rab" class="card-header"
  • heading a data-toggle="collapse" data-target="#id"
  • div class="collapse show" id="id" data-parent="#accordion"
  • div class="card-body"

Tooltips

  • tooltips: displays more information when hovering over an element
    • data-toggle="tooltip" & title="content in tooltip"
  • popovers: displays more information when clicking on an element
    • data-toggle="popover"
  • uses popper.js- library for tooltips, popovers, modals, and dropdowns
  • need to add jquery to initialize tooltips
    • <script>
      $(function () {
      $('[data-toggle="tooltip"]').tooltip()
      })
      </script>
  • then build tooltips using custom data attributes
  • to add HTML to tooltip: data-html="true"
  • data-placement="top, left, right, or bottom"
