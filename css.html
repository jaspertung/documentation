CSS Basics

CSS Rules

  • almost everything in CSS follows this basic pattern
  • selector {
    property: value;
    }

CSS Styles

  • Inline styles- write styles directly inline on each element (bad b/c can't duplicate easily)
  • <style> element- write styles in head (bad b/c doesn't translate over multiple documents)
  • External stylesheet- write styles in .css file then link (best!)

Link Element

  • Link element- goes in head of html sheet
  • <link rel="stylesheet" href="style.css">

Color

  • color property- change text color
  • background-color property- change background color
  • CSS Colors
    • Named colors: white, teal, fuschia (easiest, but limited- 140 named colors)
    • RGB: red, green, blue channels range from 0 to 255
      • rgb(255,0,0)
      • rgb(173,20,219)
      • rgb(255,255,255)
    • Hex: rgb 0-255, but represented with hexadecimal
      • #5FCFFC
      • #900C3F
      • #c63 is same as #cc6633

Text

  • text-align- determines how text is aligned horizontally within an element (center, left, right)
  • font-weight- sets weight or boldness of the font (determined by font type)
    • lighter, normal, bold, bolder
    • 100 (lighter), 400 (normal), 700 (bold), 900 (bolder)
  • text-decoration- sets the appearance of decorative lines on text (underline, linethrough)
    • can add color to line
    • can add style to line (double, dotted, wavy)
    • can get rid of default text decoration (anchor tags)
  • line-height- sets height of line box
  • letter-spacing- sets horizontal spacing between text

Font Size

  • Relative vs Absolute units
    • relative (EM, REM, VH, VW, %)- end result will change depending (better for responsive websites)
    • absolute (PX, PT, CM, IN, MM)- end result size doesn't depend on other elements
  • font-size: 20px;

Font Family

  • depends on fonts built into browser (web-safe- can assume most users have in their system)
  • add font stack- list of fonts in priority order for back ups
    • font-family: 'Times New Roman', Times, serif

CSS Selectors

  • Universal selector- selects everything (* {})
  • Element selector- selects everything of a certain type (button, headings, p)
  • Selector list- combine multiple selectors in a list (h1,h2)
  • ID selector- select only one thing (#ID {})
  • Class selector- select grouped elements with specified class(.class {})
  • Descendant selector- select all descendants within an element (li a {})
    • all anchor tags nested within list elements
    • can combine using class selectors in descendant selector
  • Adjacent selector- select only the elements immediately after the first element and in same parent element (h1 + p {})
  • Direct child selector- selects only direct children elements of the first element (footer > a {})
    • footer
      ul
      li a
      li a
      /ul
      a
    • only the a not in list is selected beause direct child of footer and not nested in ul
  • Attribute selector- selects elements with certain attributes (input[type="text"] {})
    • can choose class, id, input

Psuedo Classes

  • Keyword added to a selector that specifies a special state of the selected element(s)
  • always starts with a colon
  • Common examples: :active, :checked, :first, :hover, :not(), :nth-child()
  • :hover- change when mouse hovers (button color, underline link) (.post a:hover {})
  • :active- change when active (button pressed, link pressed) (.post button:active {}
  • :nth-of-type()- changes elements based on position (highlight every other post) (.post:nth-of-type(2n))

Psuedo Elements

  • Keyword added to a selector that lets you style a particular part of a selected element(s)
  • always starts with 2 colons
  • Common examples: ::after, ::before, ::first-letter, ::first-line, ::selection

CSS Cascade

  • the order your styles are declared in stylesheet and linked to HTML doc matters
  • later commands overwrite previous commands

Specificity

  • how the browser decides which rules to apply when multiple rules could apply to the same element
  • more specific selectors "win"
  • ID > class, attribute, pseudo-class > element, pseuo-element

Not Recommended

  • Inline styles: more specific than ID, class, and element
  • !important: most specific, not common, only use on page-specific CSS that overrides foreign CSS (external library)

Inheritance

  • Some property values set on parent elements are inherited by their child elements
  • ex. if set color on an element, every element inside it will have that color, unless a different color was applied directly to them

CSS Box Model

Width and Height

  • use width and height to change content box size
  • can use px, em, %, auto (centers)

Border and Border Radius

  • border-width: thin, thick / 4px, border-color, border-style: solid, dashed, dotted -> need to have all 3 together (order doesn't matter)
  • shorthand: combine all 3 as border: width style color
  • adding borders increases the total width and height of the content box
    • unless use box-sizing: border-box, then box size will stay the same and border subtracts from box
  • can change specific sides (border-left-width, border-top-color)
  • border-radius: to round corners of borders (px or %)
    • can make circle border with 50%

Padding

  • space inside an elements border
  • padding-left, padding-right, padding-bottom, padding-top
  • shorthand: padding
    • set all 4 at once, vertical|horizontal, top|horizontal|bottom, top|right|bottom|left

Margin

  • space outside of elements border separating it from other elements
  • shorthand works same way as padding
  • can use auto to automatically center element within its container

Display Property

  • can change block elements to inline or vice versa
  • inline value: width & height are ignored, margin & padding push elements away horizontally but not vertically

    • Lorem ipsum dolor sit, amet consectetur adipisicing elit. I AM SPAN! Officiis, nostrum?

  • block value: block elements break the flow of a document. width, height, margin, and padding are respected

    • I AM H1

      ME TOO

  • inline-block value: behaves like an inline element except width, height, margin, & padding are respected
  • none value: element still present, just not displayed (useful for JS)

CSS Units (revisited)

  • percentage- always relative to some other value (sometimes from parent or a value of the element itself)
    • width: 50% half the width of the parent (like container)
    • line-height: 50% half the font-size of the element itself
  • em- font size of the parent (for typographical properties like font-size) and font size of the element itself (for other properties like width)
    • font-size: 1em is the font-size of the parent so 2em is twice the font-size of the parent

      • I am h4 and 0.8em

        I am p and 2em

    • other properties: 1em is equal to the computed font-size of the element itself
    • issue with em: they can stack as there is more nesting and parents (ul inside ul), but good when component is reusable and want to scale it
  • rem (root ems): relative to root html element's (<html lang="en">) font-size
    • rather than scaling from parent element, scale from root element
    • if root font-size is 20px, 1rem is always 20px and 2rem is always 20px

Other useful CSS properties

Alpha Channel & Opacity

  • Alpha Channel: from 0 (transparent) - 1 (opaque)
    • #00cca00 00 (transparent) to FF (opaque)
    • rgba(255, 255, 255, 0.7)
    • Only applies to background colors of elements
  • Opacity: from 0 (transparent) - 1 (opaque)
    • Applies to all of element's contents and descendants

Position

  • Sets how an element is positioned in a document, top, right, bottom, left properties determine the final location of positioned elements
  • Examples
  • static- positioned according to normal flow of document (default)
  • relative- positioned according to normal flow, then offset relative to itself
  • absolute- not positioned according to normal flow, relative to closest positioned ancestor if none, then to initial containing block
  • fixed- positioned relative to intial containing block- stay even while scrolling (nav bar)
  • sticky- starts not fixed, but then becomes fixed later (nav bar)

Transitions

  • can specify property name, duration, timing function, delay
  • transition: background-color 1s 1s;
  • should single out properties you want to transition
  • timing functions: linear, steps, ease in, ease out, cubic-bezier
    • linear
    ease in
    steps
    cubic-bezier

Transform

  • rotate(): rotates around a fixed point on 2D plane
    • rotateX(), rotate3d(), rotateY(), rotateZ()
    • transform: rotate(45deg), transform: rotateX(0.5turn)
  • scale(): change the size of an element
    • scaleX(), scale3d(), scaleY(), scaleZ()
    • transform: scale(0.5), transform: scaleY(1.5)
  • translate(): moves an element on 2D plane
    • translateX(), translate3d(), translateY(), translateZ()
    • transform: translate(-200px, 50px), transform: translateZ(20em)
  • skew(): skews an element
    • skewX(), skewY()
    • transform: skew(15deg, 15deg), transform: skewX(0.25turn)
  • transform-origin: sets the origin for an item's transformation (bottom right, 50px 50px)
  • can combine- transform: translateX(-500px) rotate(0.5turn) scaleY(1.5);
  • applies to everything in container

Responsive CSS & Flexbox

Flexbox

  • one-dimensional layout method for laying out items in rows or columns
  • flex because can distribute space dynamically across elements of an unknown size

Flex-Direction

  • display: flex; (fit to container)
  • main axis: defined by the direction set by the flex-direction property
    • flex-direction: row; (default- left to right)
    • flex-direction: row-reverse; (right to left)
    • flex-direction: column (default- top to bottom)
    • flex-direction: column-reverse; (bottom to top)
  • cross axis: axis perpendicular to the main axis

Justify-Content

  • defines how items along the main-axis of a flex container are distributed
  • flex-start- from the start of main axis
  • flex-end- from the end of main axis
  • center- center along main axis
  • space-between- space between elements but not on outside edges
  • space-around- same space around all elements
  • space-evenly- even spaces between all elements
