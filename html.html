Semantic/Meaningful Markup

  • not calling everything div -> gives more info and more organized and accessible
  • <main>- unique to the document, not repeated on other pages (not logos, nav)
  • <nav>- anything that provides navigation links
  • <section>- standalone section (heading plus paragraph)
  • <article>- self-contained composition (forum post, article, blog entry)
  • <aside>- portion that is indirectly related to main content (sidebar)
  • <header>- introductory and navigational content
  • <footer>- ending stuff with copyright, author info, contact
  • <time>
  • <figure>- can include <figcaption>

Attributes

  • Attributes- additional values that change elements in a certain way
  • Common examples: form, href, placeholder, required, src, type, value
  • Format- key value pair- name="value"
  • Input
    • <input type="_">
  • Images
    • <img src="link or file" alt="description of image>
  • Links
    • <a href="url"> Link Text </a>
    • Absolute- need to use http:// url
    • Relative- looks for location of the file relative to the page where the link appears
      • no path provided: in the same folder as the original page
      • <a href="file.html"> Link Text </a>
      • path provided (/): in a subfolder of the current folder
      • <a href="folder/file.html"> Link Text </a>
      • (../): in a folder above the current folder
      • <a href="../file.html"> Link Text </a>
    • Boolean attributes- attribute that only has a name and no value
      • ex. <p id="welcome" hidden>
      • hidden is a boolean attribute that hides the element from view

Forms

  • <form action="url specifying where to send form data" method="specifies which HTTP request type (get/post)">
    1. Get request- searching/retrieving data (ex. google search)
    2. Post request- sending data (ex. sign up)
  • <input type="_"> creates interactive controls
    • type= text, password, date, color, file, checkbox...
  • add name attribute in input tag so when data is sent out it is properly labeled
    • to match Google search, name="q"
  • Form Label: include for in label and id in input
  • Button- needs to be within form with an action specified
  • Radio Button Input
    • <input type="radio" name="_" id="_" value="_">- only 1 choice (M,F, other)
      • need to add value to clarify data for server
      • Do you prefer dogs or cats?
      • under the name (petchoice) store the value (DOGS)
  • Checkbox Input
    • <input type="checkbox" id="_" name="_">- multiple choices
  • Select Tag
    • What's your current mood?
  • Range Input
    • <input type="range" id="_" name="_" min="_" max="_" step="_">
  • Number Input
  • Textarea Tag
  • Form Validations
    • presence validation- makes sure something is there (required, minlength)
    • data type/format (email- @something.com)
