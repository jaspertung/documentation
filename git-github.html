Git & Github

  • Git- free and open source version control system
  • Github- website to host repositories online

Git Commands

  • clone: bring repository hosted on Github into a folder on local machine
  • add: track your files and changes in Git
  • commit: save your files in Git
  • push: upload Git commits to a remote repository (Github)
  • pull: download changes from remote repository to local machine (opposite of push)
  • cd: change directory
  • mkdir: make new directory
  • .gitignore- creates file that won't be stored in remote repo
    • in new file, type any file/folder that you don't want included
    • ../
  • ls -la: see hidden folder to make sure folder is grom Github
    • folder will show as .git
    • stores commits history
  • q: exit
  • status: shows all files that were updated, created, deleted but not saved into commit yet
    • always good to check status before commit
  • -u: set upstream to say where to push on default
  • branch: check branches and star shows the one you are currently in
  • checkout: change between branches
    • checkout -b branch-name: creates new branch and moves to that branch
  • git diff: shows what changes have been made between 2 versions of code
  • git reset: undo an add or commit
    • git reset file-name
    • undo commit (git reset HEAD
  • git log: see log all your commits

Github Workflow

Write code → Commit changes → Make a pull request

Local Git Workflow

Write code → Stage changes (git add) → Commit changes (git commit) → Push changes (git push) → Make a pull request

From repo to local machine

Cloning repo

  • git clone (SSH key from Github)

Saving local changes in Git

  • check git status → will show "Modified files" so need to stage before saving to Git
  • git add
    • git add. → stage all files/folders listed in git status (untracked and modified files/folders
    • git add file.html → stage specified file/folder

Creating new file/folder locally

  • check git status → will show "Untracked files" so need to track before saving to Git
  • git add
    • git add. → track all files/folders listed in git status (untracked and modified files/folders)
    • git add file.html → track specified file/folder

Commit Locally

  • save
  • git commit -m "title -m "description"
    • git commit -m "Added index.html" -m "some description"
  • only works for modifying files not adding
    • git commit -am "title" -m "description"
    • adds and commits at the same time

SSH keys

  • SSH key to prove to Github that you are the owner of your account (connecting local machine to Github account)
    • keyname (private key) and keyname.pub (public key)
    • use private key to prove you own the public key
  • add SSH key to ssh-agent

Push to remote repository

  • git push origin master
    • origin → stands for location of Git repository
    • master → branch we want to push to
  • code changes will show on Github

Starting Repo locally

  • create new folder in VS Code or Finder
  • inititally isn't Git repo
  • git init
    • initialize to empty Git repo
    • can check with ls -la
  • git add
  • git commit

Push to remote repository

  • git push origin master doesn't work because no repo in Github to connect to
  • create new empty repo in Github
    • copy SSH
  • git remote add origin (paste SSH)
    • add a reference to the remote repo on Github
  • git remote -v → checks any remote repos connected to local repo
  • git push origin master
    • set up shortcut
      • set upstream (-u) → where to push on default
      • git push -u origin master
      • now can use just git push

Branching

  • git branch (shows branches and star shows the one you are currently in)
  • git checkout (change between branches)
    • git checkout master
    • can hit tab to auto-fill branch name
  • git checkout -b (create new branch and move into that branch)
    • git checkout -b feature-readme-instructions

Making changes in branch and committing them

  • VS Code will show "M" next to file to show it has been modified
  • git add
  • git commit
  • only saved on the branch (not on main)
  • need to commit branch before switching branches or else will be overwritten

Merge branch

  • git diff: shows what changes have been made between 2 versions of code
    • git diff branch-name
    • hit q to exit
  • push changes on branch to Github then create pull request
  • git push
    • git push -u origin branch-name

Create a pull request

  • request to have code pulled into another branch
  • pr from branch-name to master branch
  • anyone can review code, comment, make edits & can change pr by making commits
  • once merged, delete branch

Pulling to local master branch

  • git pull origin master
  • then will be able to see changes in VS Code

Deleting branch

  • git branch -d branch-name

Merge Conflits

  • multiple people trying to change the same files and Github doesn't know what to keep/delete, so need to do it manually
  • git merge master- keep branch up to date on local machine if master is changing
    • Example: update same line in master and branch
    • terminal will show merge conflict (can fix in Github, terminal, or code)
      • easiest to change in VS Code
  • then need to make another commit because fixed merge conflict
    • git commit -am "updated with master"

Undoing in Git

  • if accidentally add in Git, can undo
    • git reset OR git reset file-name
  • if accidentally commit in Git, can undo
    • git reset HEAD~1
    • HEAD: pointer to last commit
    • ~1: points 1 commit further
    • unstaged and uncommitted
  • if want to undo multiple commits or one that is further back
    • check git log for past commits git log --oneline
    • copy the commit hash
    • git checkout (hash) (file name)
    • git reset (hash)
    • unstages any changes made to file after selected commit
      • will still show in VS Code, but not in Github
  • to undo changes to commits
    • git reset HEAD (file name)
    • git checkout -- index.html
  • if want to undo everything after a certain point
    • get hash from git log
    • git reset --hard (hash)
    • not just unstage, but completely remove
